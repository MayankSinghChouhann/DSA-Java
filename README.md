## About

This repository is my complete DSA preparation log, built from the ground up in Java. Every problem here is solved with a focus on **reasoning first, code second** — starting from the brute-force approach and working toward the optimal solution, the way it's expected to be explained in a real interview.

This is not a dump of solutions copied from the internet. Each entry reflects my own understanding of the problem, the trade-offs between approaches, and the edge cases an interviewer would probe. The goal is to think like an engineer solving a problem — not like someone recalling a memorized answer.

**Target companies:** Microsoft, Google, Amazon, Apple, Netflix, Adobe, Atlassian, Uber, Flipkart, Walmart Global Tech, Oracle, Salesforce, Intuit, ServiceNow, Goldman Sachs, Morgan Stanley

## Why This Repository

FAANG/MAANG-level interviews test more than whether you can produce a working solution — they test how you *think*. This repository is built to demonstrate exactly that:

- **Pattern recognition over memorization** — problems are grouped by underlying technique (two pointers, sliding window, graph traversal, etc.) so the same mental model transfers across new, unseen problems
- **Complete decision trail** — brute force → optimization → final solution, mirroring how a candidate is expected to talk through a problem live in an interview
- **Complexity analysis as a habit**, not an afterthought
- **Consistency as a signal** — regular commits reflect discipline, which is itself something interviewers and hiring managers notice on a GitHub profile
